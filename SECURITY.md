# Security Policy

## Supported Phase
This repository currently contains architecture and requirements documentation only.

## Reporting a Vulnerability
If you discover a security issue in repository content or operational guidance:
1. Do not disclose publicly.
2. Report to the security owners through internal secure channels.
3. Include affected document, issue description, and potential impact.

## Security Baseline Requirements
- Zero-trust principles and least privilege.
- Tenant isolation requirements are mandatory.
- Secret references only; no plaintext secret material.
- Auditability and traceability for security-impacting decisions.

## Security Review Triggers
A security review is required when changes affect:
- Authentication or authorization architecture.
- Secrets management policies.
- Data retention, backup, or disaster recovery controls.
- Logging, monitoring, or audit requirements.

## Coordinated Disclosure
All findings are triaged by the security owner and tracked through internal governance workflow with severity classification, remediation deadlines, and closure evidence.
