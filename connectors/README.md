# connectors

## Purpose
Hosts connector specifications and connector-pack documentation for database, file, API, and cloud systems.

## Responsibilities
- Maintain connector capability matrix and limitations.
- Define authentication and connectivity requirements per source type.
- Publish resilience and retry expectations.

## Ownership
Data Connectivity Team

## Coding Standards
- No implementation code in this folder unless approved by Architecture Review Board and Product Governance.
- Changes SHALL align with approved architecture and SRS requirement IDs.
- Naming SHALL follow repository naming standards and semantic versioning policies.
- Security, privacy, and tenant isolation requirements SHALL be reflected in all artifacts.
